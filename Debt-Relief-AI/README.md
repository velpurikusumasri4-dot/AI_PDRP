# AI Powered Debt Relief & Financial Recovery Platform

An enterprise-grade, modern, and scalable FinTech web application to help borrowers manage debt, analyze financial health, predict settlement possibilities, and generate AI-powered negotiation letters using Google Gemini API.

## Features
- **Loan Management**: Add, edit, delete, and view all active loans.
- **Dashboard**: Visual representations (Chart.js) of your financial health, debt-to-income ratio, and monthly surplus.
- **Settlement Predictor**: Rule-based engine determining likelihood of settlement and recommended amounts.
- **AI Negotiation Generator**: Generates professional, customizable letters to banks using Google Gemini API.
- **PDF Export**: Download AI-generated letters directly as PDFs.

## Architecture
- **Frontend**: React.js, Vite, Tailwind CSS, Framer Motion, Chart.js, React-Toastify
- **Backend**: FastAPI, SQLAlchemy ORM, SQLite, Pydantic, Passlib (JWT Auth)
- **AI Integration**: Google Generative AI (Gemini Pro)

## Technology Stack
- **Frontend**: React 18+, Tailwind CSS, Axios, React Router v6
- **Backend**: Python 3.12+, FastAPI, Uvicorn, SQLite
- **Deployment**: Vercel (Frontend), Render (Backend)

## Installation & Setup

### Prerequisites
- Node.js & npm
- Python 3.12+

### Backend Setup
1. `cd backend`
2. Create virtual environment: `python -m venv venv`
3. Activate virtual environment.
4. Install dependencies: `pip install -r requirements.txt`
5. Copy `.env.example` to `.env` and fill in `GEMINI_API_KEY`.
6. Run server: `uvicorn main:app --reload` (Runs on http://localhost:8000)

### Frontend Setup
1. `cd frontend`
2. Install dependencies: `npm install`
3. Run dev server: `npm run dev` (Runs on http://localhost:5173)

## Environment Variables
- **Backend (.env)**
  - `SECRET_KEY`: JWT secret key
  - `ALGORITHM`: HS256
  - `ACCESS_TOKEN_EXPIRE_MINUTES`: 1440
  - `GEMINI_API_KEY`: Your Google Gemini API Key
  - `DATABASE_URL`: sqlite:///./debt_relief.db

## API Documentation
The API documentation is automatically generated by FastAPI.
Once the backend is running, visit: `http://localhost:8000/docs` to view the Swagger UI.

## Deployment Steps
- **Backend (Render)**: Use the `render.yaml` configuration to deploy directly from GitHub.
- **Frontend (Vercel)**: Import the `frontend` folder into Vercel. Ensure `vercel.json` handles rewrites for React Router.
